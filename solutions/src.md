# Problem sources

Ordered from absolute beginner (no C++ knowledge) to ICPC team level. Attempt sources in this
sequence. Do not skip any step.

---

## C++ syntax (before any competitive programming)

1. learncpp.com -- chapters 1-8 (read, not solved)
1. Competitive programmer's handbook, chapter 1 -- free at <https://cses.fi/book.pdf>

---

## First problems (introductory)

1. CSES problem set -- introductory problems (19 problems)
   <https://cses.fi/problemset/>
1. AtCoder beginner contest (ABC) -- problems A and B only
   <https://atcoder.jp/contests/>

---

   <https://codeforces.com/>

## Reference material (not problem sources -- use alongside all stages)

- cp-algorithms (e-maxx-eng) -- algorithm reference and implementations
  <https://cp-algorithms.com/>
- Competitive programmer's handbook (CPH) -- Antti Laaksonen, free PDF
  <https://cses.fi/book.pdf>
- Competitive programming 4 (CP4) -- Halim and Halim (paid book)
- KACTL -- KTH algorithm competition template library (study at stage 4 and above)
  <https://github.com/kth-competitive-programming/kactl>
- Introduction to algorithms (CLRS) -- for algorithm correctness and complexity proofs
- The algorithm design manual -- Skiena, for problem-solving intuition

---

## Notes

- LeetCode is not listed. It is not useful for ICPC preparation beyond the first two weeks of C++
  syntax practice.
- Use cp-algorithms as a reference while solving, not as a reading exercise.
- After each problem, commit your solution to `/solutions/<platform>/<problem-id>.cpp` and update
  your template library if the problem taught a reusable pattern.
- KACTL should inform the structure of your own team notebook once you reach stage 5.
